================================================================================
                              🚀 快速部署步骤
================================================================================

方法一：使用自动化脚本（推荐）
--------------------------------------------------------------------------------
1. 在 GitHub 上创建新仓库（例如：yitu-demo）

2. 运行初始化脚本：
   ./init-github.sh

3. 按提示输入：
   - GitHub 用户名
   - 仓库名称
   - 是否立即推送（选择 y）

4. 配置 GitHub Pages：
   - 访问仓库 Settings → Pages
   - Source 选择 "GitHub Actions"

5. 等待部署完成，访问：
   https://你的用户名.github.io/仓库名/


方法二：手动配置
--------------------------------------------------------------------------------
1. 修改 vite.config.js 中的 base 路径为您的仓库名

2. 初始化 Git 并推送：
   git init
   git add .
   git commit -m "初始化项目"
   git remote add origin https://github.com/用户名/仓库名.git
   git push -u origin main

3. 在 GitHub 仓库设置中启用 GitHub Pages（Source: GitHub Actions）

4. 等待自动部署完成

GitHub Actions 会自动构建和部署！

================================================================================
                              ⚙️ 重要配置说明
================================================================================

1. vite.config.js 中的 base 路径
   - 必须与 GitHub 仓库名一致
   - 格式：'/仓库名/'
   - 当前设置：'/yitu-demo/'

2. GitHub Actions 工作流
   - 文件位置：.github/workflows/deploy.yml
   - 触发条件：推送到 main 分支
   - 自动执行：安装依赖 → 构建 → 部署

3. 分支要求
   - 默认使用 main 分支
   - 如果使用 master 分支，需修改 deploy.yml

================================================================================
                              🐛 常见问题
================================================================================

问题1：页面显示 404
解决：检查 base 路径是否与仓库名一致

问题2：样式加载失败
解决：确认 vite.config.js 配置正确

问题3：Actions 权限错误
解决：Settings → Actions → General → Workflow permissions
      选择 "Read and write permissions"

问题4：推送失败
解决：检查远程仓库地址和推送权限
